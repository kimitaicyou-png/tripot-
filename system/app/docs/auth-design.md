# 認証・権限設計書

## 認証方式
- Google OAuth（NextAuth v5）
- ホワイトリスト制：許可されたGmailのみログイン可能
- 招待制：owner が設定画面からGmailを入力 → 即ログイン可能に

## ロール（3層）

| ロール | 説明 | アカウント管理 |
|---|---|---|
| owner | システムオーナー（小野崇） | アカウント発行・削除・権限付与 |
| manager | マネージャー（柏樹・犬飼） | × |
| member | メンバー（和泉・市岡） | × |

※ 土岐（COO）はコードレベルで全権限。UIロールとしては owner 相当で登録。

## ページアクセス

| ページ | owner | manager | member |
|---|---|---|---|
| 個人ダッシュボード（自分） | ○ | ○ | ○ |
| 個人ダッシュボード（他人） | ○ | ○ | × |
| アタックリスト | ○ | ○ | ○（自分の） |
| 案件管理 | ○ | ○ | ○（自分の） |
| 制作管理 | ○ | ○ | ○ |
| 顧客一覧 | ○ | ○ | ○ |
| 週次サマリー | ○ | ○ | × |
| 月次レポート | ○ | ○ | × |
| 事業計画 / 予算 | ○（変更可） | ○（変更可） | × |
| 設定 / アカウント管理 | ○ | × | × |

## 招待フロー

```
owner が設定画面を開く
→ 「メンバー招待」にGmailを入力
→ ロールを選択（manager / member）
→ 保存
→ 相手が /login からGoogleログインできるようになる
```

## メンバー管理の保存先

ホワイトリスト（Gmail + ロール）は localStorage ではなく
Git管理の JSON ファイルまたは auth.ts に直書き。
理由：全端末で同期が必要、DB不要の規模（5〜10人）。

将来10人超えたらVercel KV等を検討。

## 初回ログイン

- オンボーディングモーダル（3ステップ）を表示
- localStorage に `tripot_onboarded_{memberId}` フラグ
- 2回目以降は表示しない
