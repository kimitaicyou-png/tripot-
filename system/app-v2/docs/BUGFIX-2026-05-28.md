# tripot v2 バグ修正トラッキング（2026-05-28 AI デバッグ起点）

> 隊長が Claude にブラウザ直操作させてデバッグした結果（粗い、誤検知混在の可能性あり）。
> 各項目は「再現確認 → 原因特定 → 修正 → 検証」で裏取りしながら対応する。鵜呑み禁止。
> 達成設計として、横断する共通根で束ねて、優先度順にバッチで対応する。

## 優先度の考え方

- **P0 クラッシュ**：500 / 503、画面が開けない → 使えない。最優先
- **P1 集計・整合性**：KPI / 確度 / 件数が画面間でズレる → 経営判断を誤らせる
- **P2 データ汚染防止**：必須バリデーション抜け、空登録
- **P3 UX フィードバック**：処理中表示なし、成功/失敗不明、二度押し
- **P4 表示統一**：日付フォーマット、通貨、単位
- **P5 体裁・親切**：空状態の案内文、開発メモ整理、文言固定
- **P6 設計漏れ（機能追加）**：案件作成に顧客欄、タスクに案件紐付け欄 → 隊長確認推奨
